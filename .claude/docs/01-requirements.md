# 要件定義

## 1. プロジェクト概要

**melyuh-blog** は、Astroで構築する個人技術ブログ兼ポートフォリオサイトである。
単なる技術ブログではなく、「書類選考を突破するための技術的証拠」として機能させることを主目的とする。

## 2. 目標

- 手を動かしていることの証明（ブログ自体のコードも含めて）
- 技術の深さと広さの可視化
- 「社内だけで通用する人」ではないことの対外的な提示
- LinkedInへの掲載を通じた採用担当者へのアピール
- ブログを書くことを通じて、自分が何者であるかを定義していく

## 3. ターゲット

| 対象 | 目的 |
|---|---|
| 自分自身 | 学習記録・思考の整理・自己定義 |
| 転職活動の採用担当者 | 技術力・実務経験・人物像の提示 |

収益化は考えない。

## 4. 対象外（Non-goals）

- 収益化
- コメント機能（将来的に実装する可能性あり）
- ページネーション（当面）
- 推定読了時間
- 旧カスタムコンポーネント3本（LinkCard・GistCodeblock・XCard）の移植
- 記事の多言語化（UIの多言語化対応設計は行う）

## 5. ページ構成

ナビゲーション順: **HOME → BLOG → WORKS → PROFILE**

| ページ | 役割 |
|---|---|
| HOME | 第一印象・ブログの顔 |
| BLOG | 記事一覧（タグフィルタ付き） |
| WORKS | 制作物・プロジェクト一覧 |
| PROFILE | 自己紹介・経歴 |
| 記事詳細 | 各ブログ記事（動的ルーティング） |

## 6. デザイン

- **カラー**:
  - Primary（ライト）: `#00838F` / Primary（ダーク）: `#26C6DA`
  - Accent（ライト）: `#26C6DA` / Accent（ダーク）: `#00838F`
- **テーマ**: ダーク/ライトモード切替
- **Favicon**: 既存画像（`melyuh_favicon.png`）を流用
- **サイト名**: 「知識を融かして、かたちにするブログ」

## 7. 機能要件

### 7.1 ブログ

| 機能 | 詳細 |
|---|---|
| タグ | 記事にタグをつけてBLOGページでフィルタリング |
| RSSフィード | 記事更新を購読可能にする |
| 関連記事 | 記事末尾に同タグの記事を表示 |
| 検索 | サイト内全文検索 |

### 7.2 WORKSページ

プロジェクト1件あたりの情報:

| 項目 | 備考 |
|---|---|
| プロジェクト名 | |
| アプリへのリンク | |
| 説明文 | 軽い紹介文 |
| 使用技術 | タグ形式 |
| GitHubリンク | |
| 画像 | OGP等から自動取得できる場合は自動取得 |
| カテゴリ | 例: 個人開発・副業・大学 |

## 8. 非機能要件

| 要件 | 優先度 | 備考 |
|---|---|---|
| OGP対応 | 必須 | LinkedInシェア時にカード表示 |
| モバイル対応 | 必須 | レスポンシブデザイン |
| ページ速度 | 将来 | Core Web Vitals対応、後から追加 |
| SEO | 将来 | 後から追加可能な設計にする |

## 9. コンテンツ

- **記事形式**: MDX（Markdownの中にAstroコンポーネントを直接使用可能）
- **コンテンツ範囲**: 技術・非技術・趣味・人生観など幅広く

## 10. デプロイ構成

- **ホスティング**: AWS S3 + CloudFront
- **IaC**: Terraform
- **目的**: AWSおよびIaCの理解深化のためあえてAWSを採用

## 11. 言語

- **現在**: 日本語のみ
- **将来**: 英語対応可能な設計（UIテキストはコンポーネント内にベタ書きせず一箇所に集約）
