## 4. BLOGページ（記事一覧）

- **表示形式**: カードグリッド
- **タグフィルター**: ページ内でその場フィルタリング（URLは変化しない）

### カード1枚の内容

| 要素 | 取得方法 |
|---|---|
| サムネイル画像 | frontmatterの`thumbnail`、省略時はデフォルト画像 |
| タイトル | frontmatterの`title` |
| 投稿日 | frontmatterの`date` |
| タグ | frontmatterの`tags` |
| 概要文 | frontmatterの`description`、省略時は本文冒頭から自動抽出 |

---

## 5. 記事詳細ページ

### レイアウト

| 要素 | 配置・詳細 |
|---|---|
| タイトル・日付・タグ | 記事上部 |
| 目次（TOC） | MDXの見出し（`##`・`###`）から自動生成 |
| 本文 | MDXコンテンツ |
| 関連記事 | 記事下部。同タグの記事を新しい順に最大3件表示 |

**URLパターン**: `/blog/YYYY-MM-DD-title`（ファイル名全体がスラッグになる）
例: `2026-06-26-astro-migration.mdx` → `/blog/2026-06-26-astro-migration`

### フロントマター

```yaml
---
title: "記事タイトル"           # 必須
date: 2026-06-26               # 必須
tags: ["AWS", "Astro"]         # 必須
description:                   # 省略時は本文冒頭から自動抽出
thumbnail:                     # 省略時はデフォルト画像
draft: false                   # trueにすると下書き（非公開）
---
```

`title`・`date`・`tags` の3フィールドのみ記載すれば公開できる。

> フロントマターの詳細なバリデーションルール（型・必須/任意・デフォルト値）は `src/content/config.ts` が真実の源泉。このドキュメントはその意図を記録したものであり、乖離している場合はコードを優先すること。

## 6. WORKSページ（制作物一覧）

### 表示形式

カードグリッド形式でプロジェクトを一覧表示。カテゴリによるフィルタリングは将来対応。

### フロントマター

```yaml
---
title: "melyuh-blog"                    # 必須
description: "Astroで構築した技術ブログ"  # 必須
category: "個人開発"                     # 必須（個人開発・副業・大学）
techs: ["Astro", "TypeScript", "AWS"]   # 必須
siteUrl:                                # 省略可
githubUrl:                              # 省略可
image:                                  # 省略時はsiteUrlのOGPから自動取得。siteUrlも未設定の場合はデフォルト画像
order: 1                                # 省略時はファイル名のアルファベット順
featured: false                         # trueにするとHOMEの「注目のWORKS」に表示（最大3件）
---
```

フィールド名はcamelCaseで統一。

> フロントマターの詳細なバリデーションルール（型・必須/任意・デフォルト値）は `src/content/config.ts` が真実の源泉。このドキュメントはその意図を記録したものであり、乖離している場合はコードを優先すること。

### カード1枚の内容

| 要素 | 取得方法 |
|---|---|
| 画像 | frontmatterの`image`、省略時は`siteUrl`のOGPから自動取得、`siteUrl`も未設定の場合はデフォルト画像 |
| タイトル | frontmatterの`title` |
| 説明文 | frontmatterの`description` |
| 使用技術 | frontmatterの`techs`（タグ形式） |
| カテゴリ | frontmatterの`category` |
| リンクボタン | `siteUrl`・`githubUrl`（設定されている場合のみ表示） |

## 8. 検索

- **実装**: Pagefind（ビルド時にインデックス生成）
- **UI**: ヘッダーの検索アイコンをクリックするとフルスクリーンオーバーレイが開く
- **対象**: ブログ記事の全文（当面はブログ記事のみ。WORKSページ・PROFILEページは対象外）

---

## 9. RSSフィード

- ブログ記事の更新を購読可能
- 含める情報: タイトル・概要・投稿日・記事URL
- フッターのRSSリンクから到達可能

---

## 10. OGP

各ページにOGPメタタグを設定し、LinkedInなどでシェアした際にカード表示される。

| ページ | OGP画像 |
|---|---|
| 記事詳細 | frontmatterの`thumbnail`、省略時はデフォルト画像 |
| その他ページ | デフォルト画像（サイト共通） |

---

## 11. 多言語対応（将来）

UIテキスト（ナビゲーションラベル・ボタン文言など）はコンポーネント内にベタ書きせず、定数ファイルに集約する。記事の多言語化は現時点では対象外。
